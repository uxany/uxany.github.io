【摘要】陌生人：与圣战者对话

2023-09-21
随笔

【Youtube】Stranger: Talking to Jihadists | 3 hours Complete Edition | Chai Jing

关于柴静

柴静在简中是一个有争议的人物。曾经她非常亲近体制内，代表了体制内的一些改革派、反建制派媒体人的声音。后来出走，其作品在墙内彻底销声匿迹，只剩下了关于这个人的争议。

她的争议主要集中在代表作《穹顶之下》，以及网友翻出的早些年与丁仲礼院士的访谈，当然还有私人问题。我最迟在高中前就阅读过她的《看见》，后来看了《穹顶之下》。对她的态度也随着个人意识形态的变迁和大众对她的认知，在独立媒体人与公知间几经变化。这里不再讨论，但希望大家认识到一个人与一个问题的复杂性，也要认识到一个人和一个时代的局限性。

多年后再看到她的访谈，没想到聚焦在了一个离大多数中国人很远的话题：恐怖主义。该系列视频与前圣战者对话，试图揭露一些裹挟在恐怖主义下的问题与答案。

也许她是带有价值判断的，但不妨作为阅读材料来看看。

摘要

恐怖主义分子是什么人？

兄弟结构在恐怖主义分子里很常见。（让我想到《孟买酒店》中的恐怖分子）

并非教育因素，也无关于他们住在哪里，甚至也无关于他们的经济状况和社会地位。

其中有不少有前科的罪犯，亦有不少受过高等教育的精英，也有很多工作收入良好的人。

罪犯和精英会被极权主义的威势吸引，而群众靠宣传获得，大部分是盲从者。

恐怖主义背后的意识形态

  • 圣战分子将不符合原教旨教义的人们非人化，从而合理化杀戮。

  • 教义禁止自杀，但自杀式袭击被替换为殉道。圣战分子死后得上天堂的狂热，使得他们不惧死亡。

  • 教义禁止饮酒和婚前性行为，但很多圣战分子却酗酒、嫖妓，以至于 _“所有人都在装”_。圣战分子将其视为魔鬼的引诱和神的惩罚，并声称是由于这个糟糕的社会允许他们这样做。只有通过圣战让世界实行伊斯兰律法才可以解放自己。

  • 允诺的奖励缓解现实的性压抑。害怕与现实的异性接触却幻想一次胜利的战斗后挑选 sex slave ，天堂的模样也是七十二个 virgin 。

人如何成为极端分子？

  • 身份危机。穆斯林在世俗化国家的原子化。
  • 父权的缺失。失去父辈力量代表的秩序框架和归属，除了圣战组织。
  • 群体效应。有污点的圣战分子需要更极端来证明自己的虔诚，不同圣战组织时间为了争夺话语权也在竞争更极端。
  • 通过简单机械性的音乐、戒律洗脑。
  • 极端不是缺乏教育的结果，而是受教育的结果。温和的意识形态打不过极端的意识形态。
  • 面对内心的怀疑选择躲进阴谋论。
  • 泛滥的自由下，严苛的教义反而一种舒适区，不需要为了未来建立计划。经书为人生提供了一切答案。“自由对于无力掌握它的人来说，是沉重的负担。”

如何对抗恐怖主义？

  • 极右分子与圣战分子像一个硬币的两面。
  • 真正的转变不是找到了真理，是不再寻找终极真理，不是得到了答案，是不再需要单一的答案。不是找到了一个身份，而是拥有一个永远开放、不断更新的身份。

【个人看法】

  • 非常难得的一手材料。
  • 恐怖主义的意识形态包装非常高明，换一种包装或许在某些世俗化非常成功的国家也很有市场。 可以在纳粹反犹主义、麦卡锡主义、斯大林主义、红色高棉的大清洗中找到影子。
  • 非母文化语境下，哪怕是移民国家也永远不可能真正融入异国他乡，一代移民乃至二代移民的陌生人处境永远都在。因此移民仅作为一个提高生活质量或者没有退路的可选项，并不能作为一个实现个人价值的配置项。
  • 泛滥的多元和自由不一定是好事。保证共识前提下的多元，自我规划前提下的自由才能促进社会的进步。
  • 温和的意识形态打不过极端的，想要吸引两端的群体，只会把两端的都推向极端。从受众的角度讲，两鸟在林不如一鸟在手。
  • 某种意义上讲，亚文化群体的极端化只是时间问题，包括不限于任何广义个人崇拜向的粉圈，任何见不得光的小众爱好。当然也包括东亚特色的儒家性压抑群体。
  • 拥抱主流文化圈，拥抱现实生活才能避免极端化。（当一个现充吧）

Vue3核心语法

OptionsAPI 与 CompositionAPI

  • Vue2 API设计是Options（配置）风格的
  • Vue3 API设计是Composition（组合）风格的

setup

setup是Vue3中一个新的配置项，值是一个函数，它是 Composition API “表演的舞台”，组件中所用到的：数据、方法、计算属性、监视……等等，均配置在setup中。

setup 与 Options API 的关系:

  • Vue2 的配置（data、methods……）中可以访问到 setup中的属性、方法。
  • 但在setup中不能访问到Vue2的配置（data、methos……）。
  • 如果与Vue2冲突，则setup优先。

setup语法糖:

ref 创建：基本类型的响应式数据

reactive 创建：对象类型的响应式数据

ref 创建：对象类型的响应式数据

  • 其实ref接收的数据可以是：基本类型、对象类型。
  • 若ref接收的是对象类型，内部其实也是调用了reactive函数。

TS中的接口、泛型、自定义类型

Vue 生命周期

  • 创建前(beforeCreate) / 创建(created)

  • 更新前(beforeUpdate) / 更新(updated)

其他API

shallowRef 与 shallowReactive

通过使用 shallowRef() 和 shallowReactive() 来绕开深度响应。浅层式 API 创建的状态只在其顶层是响应式的，对所有深层的对象不会做任何处理，避免了对每一个内部属性做响应式所带来的性能成本，这使得属性的访问变得更快，可提升性能。

readonly 与 shallowReadonly

readonly

  1. 作用：用于创建一个对象的深只读副本。

  2. 用法：

  3. 特点：

    • 对象的所有嵌套属性都将变为只读。
    • 任何尝试修改这个对象的操作都会被阻止（在开发模式下，还会在控制台中发出警告）。

  4. 应用场景：

  3. 特点：

    • 只将对象的顶层属性设置为只读，对象内部的嵌套属性仍然是可变的。

toRaw 与 markRaw

toRaw

  1. 作用：用于获取一个响应式对象的原始对象， toRaw 返回的对象不再是响应式的，不会触发视图更新。

    官网描述：这是一个可以用于临时读取而不引起代理访问/跟踪开销，或是写入而不触发更改的特殊方法。不建议保存对原始对象的持久引用，请谨慎使用。

    何时使用？ —— 在需要将响应式对象传递给非 Vue 的库或外部系统时，使用 toRaw 可以确保它们收到的是普通对象

markRaw

  1. 作用：标记一个对象，使其永远不会变成响应式的。

    例如使用mockjs时，为了防止误把mockjs变为响应式对象，可以使用 markRaw 去标记mockjs

customRef

作用：创建一个自定义的ref，并对其依赖项跟踪和更新触发进行逻辑控制。

MVVM模型

  1. M：模型(Model) ：data中的数据
  2. V：视图(View) ：模板代码
  3. VM：视图模型(ViewModel)：Vue实例

数据代理

Vue中的数据代理：

  • 通过vm对象来代理data对象中属性的操作（读/写）

Vue中数据代理的好处：

  • 更加方便的操作data中的数据

基本原理：

  • 通过Object.defineProperty()把data对象中所有属性添加到vm上。
  • 为每一个添加到vm上的属性，都指定一个getter/setter。
  • 在getter/setter内部去操作（读/写）data中对应的属性。

事件处理

v-on: @

  • 事件的回调需要配置在methods对象中，最终会在vm上

事件修饰符

  1. prevent：阻止默认事件（常用）；
  2. stop：阻止事件冒泡（常用）；
  3. once：事件只触发一次（常用）；
  4. capture：使用事件的捕获模式；
  5. self：只有event.target是当前操作的元素时才触发事件；
  6. passive：事件的默认行为立即执行，无需等待事件回调执行完毕；

  • keyup
  • keydown
  • keypress

计算属性

  • 计算属性是基于它们的响应式依赖进行缓存的
  • 与methods实现相比，内部有缓存机制（复用），效率更高，调试方便。
  • 1.计算属性最终会出现在vm上，直接读取使用即可。

computed和watch之间的区别

  • computed能完成的功能，watch都可以完成。
  • watch能完成的功能，computed不一定能完成，例如：watch可以进行异步操作。
